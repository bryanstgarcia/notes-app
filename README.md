# My App

Full-stack web application with a Django REST API backend and a Next.js frontend.

## Stack

| Layer | Technology | Purpose |
|---|---|---|
| Backend | Django 5 + DRF | REST API |
| Authentication | SimpleJWT | Access + Refresh tokens |
| Frontend | Next.js 16 | App Router, SSR |
| Styles | Tailwind CSS | CSS utilities |
| Types | TypeScript | Client-side typing |
| API contract | OpenAPI / drf-spectacular | Auto-generated schema |
| Database | SQLite | Local development |
| Orchestration | Docker Compose | Start everything with one command |

## Prerequisites

- Docker
- Docker Compose

## Quick start

```bash
git clone <repo>
cd <repo>
cp .env.example .env
make build
make migrate
make dev
```

## Development URLs

- Frontend: `http://localhost:3000`
- Backend API: `http://localhost:8000/api/`
- Swagger UI: `http://localhost:8000/api/docs/`

## Available commands

| Command | Description |
|---|---|
| `make dev` | Start backend and frontend |
| `make build` | Build both Docker images |
| `make down` | Stop and remove containers |
| `make logs` | Follow logs from all services |
| `make migrate` | Apply Django database migrations |
| `make makemigrations` | Generate new Django migrations |
| `make createsuperuser` | Create a Django admin superuser |
| `make shell` | Open a Django shell |
| `make reset-db` | Wipe the SQLite volume and re-run migrations |
| `make generate-schema` | Generate the OpenAPI schema and copy it to the frontend |

## OpenAPI schema workflow

The contract between backend and frontend is the OpenAPI schema generated by `drf-spectacular` from the Django REST Framework views and serializers.

Whenever the backend API changes:

1. Run `make generate-schema` from the root. This generates `backend/openapi.json` and copies it to `frontend/openapi.json`.
2. Run `npm run generate-api` inside `frontend/` to regenerate the typed client in `frontend/src/lib/api/`.

The generated client should never be edited by hand — any manual change will be overwritten on the next regeneration.
